# IT Maintenance Fullstack App

Sistem maintenance perangkat IT berbasis React + Vite di frontend dan PostgreSQL/PostgREST yang diakses dengan `@supabase/supabase-js`. Repository ini dirancang untuk deployment self-hosted memakai Docker.

Konvensi database pada project ini:

- Custom schema yang dipakai aplikasi adalah `maintenance`
- Tabel utama ada di `maintenance.maintenance`
- RPC helper juga dibuat di schema `maintenance`

Catatan arsitektur:

- Repo ini mengirimkan stack backend ringan yang kompatibel dengan pola query Supabase (`select`, `insert`, `update`, `delete`, `eq`, `gte`, `lte`) melalui endpoint `/rest/v1`.
- Untuk kebutuhan Auth, Storage, Realtime, dan Studio penuh ala Supabase official, gunakan bundle Docker resmi Supabase sebagai pengembangan lanjutan. Auth pada spesifikasi ini saya tandai opsional dan tidak diaktifkan default di compose utama.

## Fitur yang Sudah Disiapkan

- Dashboard dengan KPI total maintenance, total durasi, kegiatan paling sering, grafik per tahun, dan grafik per jenis kegiatan.
- Halaman data maintenance dengan CRUD, pagination, search, sorting, loading state, empty state, toast, dan reset filter.
- Auth Supabase email/password untuk login teknisi/admin di frontend.
- Pengunjung anonim tetap bisa melihat dashboard dan tabel, tetapi CRUD dibatasi untuk user yang login.
- Filter dinamis:
  - Lokasi sebagai filter utama
  - Opsi tahun baru muncul setelah lokasi dipilih
  - Pemilihan tahun otomatis membentuk range `01-01` sampai `12-31`
  - Range tanggal manual akan mengosongkan filter tahun
- Export data ke Excel.
- Dark mode.

## Skema SQL

Core table:

```sql
create schema if not exists maintenance;

create table maintenance.maintenance (
  id bigint generated by default as identity primary key,
  tanggal_maintenance date not null,
  kode_aset text not null,
  nama_perangkat text not null,
  tipe text,
  lokasi text,
  jenis_kegiatan text,
  durasi integer,
  catatan text,
  created_at timestamp default now()
);

create index idx_maintenance_lokasi on maintenance.maintenance(lokasi);
create index idx_maintenance_tanggal on maintenance.maintenance(tanggal_maintenance);
```

File implementasi lengkap beserta policy dan RPC helper ada di [supabase/migrations/20260424_create_maintenance.sql](/D:/Koding/maintenance/supabase/migrations/20260424_create_maintenance.sql:1).

Perilaku auth pada SQL tersebut:

- `SELECT` boleh untuk `anon`
- `INSERT`, `UPDATE`, `DELETE` hanya untuk `authenticated` dan `service_role`
- frontend tetap memakai `ANON_KEY`, lalu setelah user login Supabase akan membawa access token user untuk operasi CRUD

## Menjalankan di Local

1. Copy environment file.

```bash
cp .env.example .env
cp frontend/.env.example frontend/.env
```

2. Isi `JWT_SECRET` pada `.env`.

3. Generate token `anon` dan `service_role`.

```bash
node supabase/config/generate-jwt.mjs your_jwt_secret_here
```

4. Masukkan output `SUPABASE_ANON_KEY` dan `SUPABASE_SERVICE_ROLE_KEY` ke `.env`.

5. Samakan `frontend/.env` dengan nilai publik berikut:

```env
VITE_SUPABASE_URL=http://localhost:8000
VITE_SUPABASE_ANON_KEY=<isi_dengan_SUPABASE_ANON_KEY>
```

6. Jalankan stack.

```bash
docker compose up -d --build
```

7. Buka aplikasi:

- Frontend: `http://localhost:8088`
- REST API: `http://localhost:8000/rest/v1/maintenance`
- Studio opsional: `docker compose --profile studio up -d`

Catatan untuk akses REST mentah:

- karena tabel berada di schema `maintenance`, request REST mentah di Supabase/PostgREST perlu profile schema `maintenance`
- client `supabase-js` pada project ini sudah mengirim schema tersebut otomatis

Jika Anda memakai Supabase official yang sudah ada, schema `maintenance` harus ikut diekspos ke API. Menurut dokumentasi resmi Supabase, custom schema perlu:

- ditambahkan ke daftar exposed schemas
- diberi `GRANT USAGE` dan grant untuk table/routine/sequence

Pada self-hosted official stack, biasanya ini berarti menambahkan `maintenance` ke konfigurasi schema PostgREST, misalnya:

```env
PGRST_DB_SCHEMAS=public,storage,graphql_public,maintenance
```


Lalu buka `http://localhost:5173`.

## Menyiapkan User Login

Karena CRUD dibatasi untuk user login, buat minimal satu user di Supabase Auth:

1. Buka Supabase Studio Anda.
2. Masuk ke `Authentication > Users`.
3. Tambahkan user email/password secara manual atau aktifkan signup sesuai kebutuhan.

Login dari frontend menggunakan akun tersebut.

## Deploy ke Server

1. Install Docker dan Docker Compose Plugin di server.
2. Clone repository ke server.
3. Buat `.env` produksi dan ganti seluruh secret default.
4. Set `SUPABASE_URL` dan `SUPABASE_PUBLIC_URL` ke URL publik gateway Anda, misalnya `https://api.domain-anda.com`.
5. Build dan start:

```bash
docker compose up -d --build
```

6. Pasang reverse proxy eksternal seperti Nginx atau Traefik untuk TLS/HTTPS.

## Integrasi Frontend

Client Supabase ada di [frontend/src/lib/supabaseClient.js](/D:/Koding/maintenance/frontend/src/lib/supabaseClient.js:1). Query utama list ada di [frontend/src/lib/maintenance.js](/D:/Koding/maintenance/frontend/src/lib/maintenance.js:1).

Client default pada project ini diarahkan ke schema `maintenance`, jadi query tabel dan RPC akan mengarah ke schema tersebut. Contoh pola query yang dipakai:

```js
supabase
  .from("maintenance")
  .select("*", { count: "exact" })
  .eq("lokasi", filters.lokasi)
  .gte("tanggal_maintenance", filters.tanggalMulai)
  .lte("tanggal_maintenance", filters.tanggalSelesai)
  .order("tanggal_maintenance", { ascending: false })
  .range(from, to);
```

Kalau Anda ingin query eksplisit per schema, Supabase juga mendukung pola resmi seperti `supabase.schema("maintenance").from("maintenance")`.

## Best Practice Penggunaan

- Gunakan filter lokasi sebelum membuka dataset besar agar query tetap ringan.
- Tetapkan `pageSize` moderat, misalnya `10` atau `20`, untuk menjaga waktu respons.
- Pin image Docker ke tag spesifik sebelum produksi jika Anda ingin deployment yang lebih deterministik.
- Ganti policy RLS default yang saat ini terbuka untuk `anon` jika Anda mengaktifkan autentikasi.
- Tambahkan reverse proxy HTTPS dan backup PostgreSQL berkala untuk deployment server.

## Pengembangan Lanjutan

- Tambahkan Supabase Auth resmi bila ingin role `admin` dan `teknisi`.
- Tambahkan Supabase Storage bila ingin upload foto perangkat.
- Tambahkan table audit/log jika ingin histori perubahan data.
